Задание 1. Расчет адресов


-Бинарная нотация IP-адреса — это представление его в двоичном виде (по основанию 2). 
IP-адрес (IPv4) состоит из 4 октетов (байтов), 
каждый из которых можно перевести из десятичного в двоичный формат.

a.)Расчитать бинарную нотацию для адресов:
1.)178.101.89.7 -> 10110010.1100101.1011001.111
    178 -> 10110010; 101 -> 1100101; 89 -> 1011001 ; 7 -> 111
2.)201.57.153.161 -> 11001001.111001.10011001.10100001
    201 -> 11001001; 57 -> 111001; 153 -> 10011001; 161 -> 10100001

b.)Выделяем подсети для различных сегментов организации:
-технологический (3 машины):
178.101.89.0 -> 2^3 - 2 = 6 -> /29 (255.255.255.248) (Чтобы количество хостов было не менее 3, необходимо выделить маску /29)
-серверный (16 машин):
178.101.89.0 -> 2^5 - 2 = 30 -> /27 (255.255.255.224) (Чтобы количество хостов было не менее 16, необходимо выделить маску /27)
-пользовательский (из 32 машин): 
178.101.89.0 -> 2^6 - 2 = 62 -> /26 (255.255.255.192) (Чтобы количество хостов было не менее 32, необходимо выделить маску /26)

c.)Выделяем подсети под сегменты: t(3 машины), s(16 машин), p(32 машины):
178.101.89.7:
Доступные адреса: 178.101.89.1 - 178.101.89.254
t:
178.101.89.0/24 -> 2^n >= 3 -> n = 2 -> /24 + 2 = /26 (255.255.255.192)
Количество хостов в подсети: 2^6 - 2 = 62
Шаг подсети: 256 - 192 = 64
Рассчет подсетей:
Подсеть	Диапазон адресов	             Network ID	     Broadcast	     Доступные хосты
1	    178.101.89.0 - 178.101.89.63	 178.101.89.0	 178.101.89.63	 178.101.89.1 - 178.101.89.62
2	    178.101.89.64 - 178.101.89.127	 178.101.89.64	 178.101.89.127	 178.101.89.65 - 178.101.89.126
3	    178.101.89.128 - 178.101.89.191	 178.101.89.128	 178.101.89.191	 178.101.89.129 - 178.101.89.190

*Количество подсетей: 4 (но нам нужно 3, одну можно зарезервировать)(Резервируем 4-ую)
178.101.89.7 Находится в первой подсети (178.101.89.0/26) -> доступные адреса в этой подсети - 178.101.89.1 - 178.101.89.62

Альтернативный вариант (если важно иметь именно 3 подсети):
Первая подсеть (большая):
/25 -> (255.255.255.128) -> хосты -> 256 - 128 - 2 (2^7 - 2) = 126
Шаг подсети: 256 - 128 = 128 -> Диапазон: (178.101.89.0 - 178.101.89.127) -> Доступные хосты: (178.101.89.1 - 178.101.89.126)
